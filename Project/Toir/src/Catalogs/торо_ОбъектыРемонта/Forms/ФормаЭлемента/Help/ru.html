Справочник «Объекты ремонта»

 

В форме выводятся следующие поля:

  • Наименование — название объекта ремонта в справочнике.
  • Код — код объекта ремонта в справочнике.
  • Направление — наименование направления объекта ремонта.
  • Типовой ОР – наименование типового объекта ремонта (ремонтной группы), в которую входит объект ремонта.

 

Закладка «Общие»

  • Объект принадлежит стороннему контрагенту — признак, устанавливаемый для объектов ремонта, которые принадлежат организации-контрагенту. Если признак установлен, то вместо полей  Организация и Подразделение отображаются поля Контрагент и Договор соответственно.
  • Организация — название организации, к которой относится объект ремонта.
  • Подразделение — название подразделения организации, к которому относится объект ремонта.
  • Контрагент — название контрагента, которому принадлежит объект ремонта.
  • Договор контрагента — наименование договора с контрагентом, которому принадлежит объект ремонта.
  • Дата ввода в экспл. (отображается, если отключена настройка "Использовать документы принятия и списания оборудования") — дата ввода в эксплуатацию объекта ремонта.
  • График работы — обозначение графика работы объекта ремонта, установленный по плану.
  • Срок полезного исп.— срок, в течение которого объект ремонта приносит экономические выгоды (доход) организации (указывается в месяцах).
  • Инвентарный № — инвентарный номер объекта ремонта.
  • Технологический № — технологический номер изделия–объекта ремонта.
  • Местонахождение — фактический адрес подразделения организации или контрагента, которому принадлежит объект ремонта.
  • Введён в эксплуатацию (отображается, если включена настройка "Использовать документы принятия и списания оборудования") — дата ввода в эксплуатацию объекта ремонта.
  • Ввести документ "Принятие объекта ремонта к учёту"  (отображается, если включена настройка "Использовать документы принятия и списания оборудования") — ссылка на создание документа, с помощью которого объект ремонта ставится на учёт.
  • Снят с учёта (отображается, если включена настройка "Использовать документы принятия и списания оборудования") — дата снятия объекта ремонта с учёта.
  • Ввести документ "Выбытие объекта ремонта" (отображается, если включена настройка "Использовать документы принятия и списания оборудования")  — ссылка на создание документа, с помощью которого объект ремонта снимается с учёта.
  • Изготовитель — название завода-изготовителя объекта ремонта.
  • Дата выпуска — дата выпуска объекта ремонта.
  • Номер паспорта — номер паспорта изделия–объекта ремонта.
  • Заводской № — номер изделия–объекта ремонта, установленный на заводе по номенклатуре объектов.
  • ОКОФ – значение из общероссийского классификатора основных фондов.
  • Подразделение-исполнитель — наименование подразделения организации, которое занимается обслуживанием объекта ремонта (указанное значение подставляется по умолчанию при оформлении документов).
  • Подрядчик ремонта — наименование подрядной организации, которая занимается обслуживанием объекта ремонта (указанное значение подставляется по умолчанию при оформлении документов).
  • Приоритет – наименование приоритета, в соответствии с которым производится обслуживание объекта ремонта.
  • Не участвует в планировании — признак того, что на объект ремонта не будут составляться графики ППР. Если флажок установлен, то объект ремонта не попадает в список для отбора при формировании графиков ППР.
  • Условие ремонта – наименование условия ремонта: останов основного оборудования или без останова основного оборудования.
  • Это актив – при установке флага, в случае использования обмена с ERP 2, объект ремонта будет перенесен в ERP 2.
  • Штрих-код – штрих-код передается из мобильного приложения "ТОиР 2: Регистратор".
  • Для объекта ремонта имеются сроки гарантии — флаг отображает наличие гарантийных периодов для объекта ремонта. Если флаг установлен и недоступен для редактирования, значит периоды гарантии были унаследованы от головного объекта.

Закладка «Показатели эксплуатации/ Показатели»

Группа полей "Контролируемые показатели" содержит перечень показателей, значения которых необходимо контролировать.

  • Показатель — наименование показателя, который должен контролироваться в разрезе текущего объекта ремонта.

  • Ед. изм. – единица, в которой измеряется показатель.
  • Из ОР – признак использования в системе по умолчанию значений контролируемых показателей, заданных для конкретного объекта ремонта. Если признак не установлен, то для объекта ремонта будет использоваться значения показателя, указанное в ремонтной группе.
  • Доп. максимум – максимальное допустимое значение контролируемого показателя.
  • Доп. минимум – минимальное допустимое значение контролируемого показателя.
  • Крит. максимум – максимальное критическое значение контролируемого показателя, при превышении которого необходим внеплановый ремонт оборудования для предотвращения его аварийного выхода из строя.
  • Крит. минимум – минимальное критическое значение контролируемого показателя. В случае, если значение контролируемого показателя ниже критического минимума, необходим внеплановый ремонт оборудования для предотвращения его аварийного выхода из строя.
  • Точка контроля – дата и время регистрации значения контролируемого показателя.

 

Подзакладка «Показатели эксплуатации/Показатели»/«Журнал ввода значений показателей»
  • № — номер документа в списке.
  • Дата контроля – дата регистрации значения контролируемого показателя.
  • Значение – зарегистрированное значение контролируемого показателя.
  • Документ – дата, время создания и название документа, регистрирующего значение контролируемого показателя.

 

Подзакладка «Показатели эксплуатации/Показатели»/«Параметры оповещения»

  • Вид сравнения – список вариантов отбора.
  • Вид значения – список видов значения контролируемых показателей
  • Значение – необходимое для сравнения значение контролируемого показателя
  • Параметры уведомления –параметры уведомлений в документах.
  • Параметры ввода дефекта - параметры автоматичекой фиксации дефектов.

 

Закладка «Показатели эксплуатации/Наработка»

Группа полей "Показатели плановой наработки" содержит перечень показателей, для которых в системе регистрируется наработка объекта ремонта.

  • Дата введения – дата введения показателя плановой наработки.
  • Показатель наработки – наименование показателя, по которому фиксируется наработка текущего объекта ремонта.
  • Ед. изм. – единица, в которой измеряется наработка объекта ремонта для текущего показателя.
  • Период наработки – промежуток времени, на который вводится плановое значение наработки. Значение выбирается из выпадающего списка и может содержать одно из предопределенных значений: День, Неделя, Декада, Месяц, Квартал, Полугодие, Год.
  • План. нараб. – значение, которое будет использоваться для расчета наработки объекта ремонта в случае отсутствия фактических значений в журнале наработки.
  • КИО – коэффициент учета плановой наработки. Коэффициент учитывает долю аварийных ремонтов при планировании по наработке.

 

Группа полей "Журнал наработки" содержит список фактических значений наработки объекта ремонта.

  • Дата работы по — дата, на которую введена наработка.
  • Наработка — значение наработки объекта ремонта.
  • Документ — дата, время создания и название документа, которым введена наработка.

 

Группа полей "Максимальные значения показателей наработки"

  • Показатель наработки — наименование показателя наработки.
  • Макс. значение — максимальное значение показателя наработки, при достижении которого счётчик наработки обнуляется.

 

Подзакладка «Показатели эксплуатации/Состояние ОР»/«Диаграмма состояния ОР»

Информация в форме представлена в виде диаграммы Ганта, которая отражает состояние оборудования во времени. По вертикальной оси отображаются виды состояний оборудования в системе. По горизонтальной оси - продолжительность во времени. Под диаграммой расположена таблица, отражающая изменение состояний объектов ремонта. В форме есть возможность отбора данных для отображения по периоду (признак [Отбор по периоду]). По умолчанию отбор осуществляется по последнему году от текущей даты.

Вводить состояние объекта ремонта можно на дату или на период (кнопка [Ввод состояния]).

Подзакладка «Показатели эксплуатации/Состояние ОР»/«Количество пусков»

  • Вид пуска — наименование вида пуска оборудования.
  • Количество — количество пусков данного вида.

 

Закладка «Нормативы планирования/ Нормативные ремонты»

  • Вид ремонта – наименование вида ремонта.
  • Технологическая карта– наименование технологической карты ремонта, в которой описывается состав и последовательность работ, их продолжительность и другие данные для определенного вида ремонта.
  • Способ планирования – наименование способа планирования, который будет использоваться при расчете графика ППР для определенного вида ремонта.

 

Подзакладка «Нормативы планирования/Нормативные ремонты»/«Предшествующие ремонты»

  • Предшествующий вид ремонта – наименование вида ремонта, который при расчете графика ППР будет являться точкой отсчета для планируемого ТО или ремонта.

 

Подзакладка «Нормативы планирования/Нормативные ремонты»/«Замещаемые ремонты»
  • Замещ. вид ремонта – наименование вида ремонтных работ, который будет замещаться текущим видом ремонта.
  • Способ замещения – название одного из предопределенных в системе способов замещения, указывающего на то, каким образом при расчете графика ППР будет производится замещение: Сдвиг вперед; Замещение; Сдвиг назад.
  • Дней от нач. – значение, указывающее на количество дней до начала выполнения замещаемого вида ремонтных работ (при попадании замещающего вида ремонта в этот период вызывается замещение).
  • Дней от оконч. – значение, указывающее на количество дней после окончания выполнения замещаемого вида ремонтных работ (при попадании замещающего вида ремонта в этот период вызывается замещение).

 

Подзакладка «Нормативы планирования/Нормативные ремонты»/«Дополнительно»

  • График ремонтных работ – наименование графика работы ремонтного персонала, в соответствии с которым производится планирование ТО и ремонтов.
  • Способ выполнения – наименование способа выполнения ремонтных работ: Хозспособ или Подрядный.
  • Способ учета даты отсчета – название способа, в соответствии с которым учитывается дата отсчета графика ППР: По дате ввода в эксплуатацию или последнему ремонту; Использовать дату планирования; По последнему ремонту; По последнему ремонту или дате планирования.
  • Не использовать в планировании – признак того, что нормативный ремонт не будет планироваться при формировании графика ППР и будет иметь справочное значение.
  • Не учитывать при планировании график работы оборудования – признак, определяющий способ расчета плановой наработки для определения даты следующего ремонта при расчете графика ППР. Если признак не установлен, то расчет плановой наработки будет выполняться с учетом графика работы оборудования, т.е., если оборудование не работает в выходные, считается, что у него в выходные наработки нет, а в остальные дни наработка будет равна указанной для объекта ремонта в закладке “Наработка. Если признак установлен, то расчет плановой наработки будет выполняться без учета графика работы оборудования, и наработка за любой день будет одинакова и равна указанной для объекта ремонта в закладке “Наработка”.
  • Не учитывать при планировании график ремонтных работ – признак того, что при формировании графика ППР ремонты будут запланированы без учета графика ремонтов оборудования, заданном в ремонтной группе объекта. Т.е., если ремонты согласно графику ремонтных работ в ремонтной группе не выполняются в выходные, то ремонты на выходные все равно будут запланированы – длительность ремонтов будет календарной.
  • Не учитывать продолжительность – признак, определяющий дату начала планирования следующего ремонта. Если признак установлен, то в качестве даты следующего ремонта определенного вида будет использована дата начала предыдущего. Если признак не установлен, то в качестве даты следующего ремонта определенного вида будет использована дата дня, следующего за окончанием предыдущего.
  • Учитывать простои оборудования – признак того, что при формировании графика ППР ремонты будут запланированы с учетом простоев оборудования.
  • Использовать коэффициенты ремонтных особенностей – признак того, что при формировании графика ППР будут учитываться заданные коэффициенты ремонтных особенностей.
  • Разрешать сокращать межремонтный период при планировании – признак перемещения планового ремонта на более раннюю дату, если дата ремонта попадает на выходные, в сторону сокращения межремонтного периода. Если перенос должен быть осуществлен на архивную дату – перемещение ремонта не выполняется. Если отметка не установлена, то при попадании следующих плановых ремонтов на выходные, при условии, что для этого вида ремонта ремонты на выходных не выполняются, начало ремонта переносится на первый рабочий день после выходных.

 

Закладка «Нормативы планирования/Фиксированные ремонтные циклы»

  • Вид цепочки– вид цепочки ремонтов.
  • Не учитывать простои – признак того, что при формировании графика ППР ремонты цепочки будут запланированы без учета простоев оборудования.
  • № – порядковый номер ремонта в цепочке.
  • Вид ремонта – наименование вида ремонта.
  • Нормативный ремонт– наименование технологической карты ремонта, в которой описывается состав и последовательность работ, их продолжительность и другие данные для определенного вида ремонта.
  • Способ планирования – наименование способа планирования, который будет использоваться при расчете графика ППР для определенного вида ремонта цепочки.
  • Способ выполнения - наименование способа выполнения ремонтных работ: Хозспособ или Подрядный.
  • График ремонтных работ - наименование графика работы ремонтного персонала, в соответствии с которым производится планирование ТО и ремонтов.

 

Закладка «Нормативы планирования/Структура ремонтного цикла»

  • Начало цикла - вид ремонта, являющийся началом и концом ремонтного цикла.

С помощью кнопки [Рассчитать и построить] можно вывести графическое представление ремонтного цикла.

С помощью кнопки [Обновить] можно выполнить перерасчет визуализации ремонтного цикла.

С помощью кнопки [Настройки] можно вывести форму выбора видов ремонтов для отображения.

 

Закладка «Нормативы планирования/Запчасти»

  • Вести учет запчастей — признак ведение учета серийных запчастей.
  • Номенклатура – наименование единицы номенклатуры, которой соответствует запасная часть.
  • Кол-во – количество единиц номенклатуры в указанных ед. измерения, необходимых для выполнения ТО и ремонтов для объекта ремонта.
  • Характеристика – характеристика номенклатуры.
  • Качество – значение, определяющее качество номенклатуры.

 

Подзакладка «Нормативы планирования/Запчасти»/«Нормативные затраты при ТОиР»

  • Вид ремонта – наименование вида ремонта, при выполнении которого используется запасная часть.
  • Кол-во – количество единиц номенклатуры в указанных ед. измерения, используемых в указанном виде ремонта.

 

Подзакладка «Нормативы планирования/Запчасти»/«История движения серийных запчастей»

  • Серийный номер – серийный номер запчастей.
  • Кол-во – количество замененных запчастей.
  • Ед. изм. – единица измерения номенклатуры.
  • Статус движения ЗЧ – вид установки: установка, замена.
  • Дата замены – дата установки запчасти.

 

Закладка «Нормативы планирования/Оценка РМ»

  • Дата – дата ввода оценки риск-менеджмента.
  • Вид ремонта – вид ремонта, для которого выполняется оценка риск-менеджмента.
  • Сезон – годовой сезон выполнения ремонтов.
  • Вероятность выхода из строя – оцениваемая вероятность выхода из строя в процентах при невыполнении данного вида ремонта
  • Ущерб – денежный эквивалент ущерба при невыполнении данного вида ремонта.

 

Закладка «Нормативы планирования/Коэффициенты ремонтных особенностей»

 

Группа полей "Рассчитываемые коэффициенты ремонтных особенностей"

  • Вид коэффициента – вид рассчитываемого коэффициента ремонтных особенностей.
  • Значение коэффициента – наименование значения рассчитываемого коэффициента ремонтных особенностей.
  • Коэф. – значение рассчитываемого коэффициента ремонтных особенностей.

 

Группа полей "Устанавливаемые коэффициенты ремонтных особенностей"

  • Вид коэффициента – вид устанавливаемого коэффициента ремонтных особенностей.
  • Значение коэффициента – наименование значения устанавливаемого коэффициента ремонтных особенностей.
  • Коэф. – значение устанавливаемого коэффициента ремонтных особенностей.
 
Итоговый коэффициент рем. особенностей – итоговое значение рассчитываемых и устанавливаемых коэффициента ремонтных особенностей.
 

Закладка «Нормативы планирования/Основное и зависимое оборудования»

  • Вид ремонта – вид ремонта, для которого настраивается зависимость с основным и/или зависимым оборудованием.

 

Подзакладка «Нормативы планирования/Основное и зависимое оборудования»/«Основное оборудование»

  • Объект ремонта - наименование объекта ремонта, который является основным для данного.
  • Вид ремонта основного объекта – наименование вида ремонта основного оборудования, с которым будет связан указанный ремонт данного объекта.

 

Подзакладка «Нормативы планирования/Основное и зависимое оборудования»/«Зависимое оборудование»

  • Объект ремонта - наименование объекта ремонта, который является зависимым данного.
  • Вид ремонта зависимого объекта – наименование вида ремонта зависимого оборудования, который будет связан с указанным ремонтом данного объекта.

 

Закладка «Журналы/ Выявленные дефекты»

  • Документ — дата, время создания и название документа с зарегистрированным дефектом.
  • Типовой дефект — наименование часто повторяющего дефекта в рамках определённого направления объектов ремонта.
  • Отказавший элемент — наименование ремонтной единицы, для которой регистрируется неисправность.
  • Вид дефекта — наименование вида дефекта.
  • Причина дефекта — название причины появления дефекта.
  • Критичность дефекта — степень критичности выявленного дефекта.

 

Закладка «Журналы/ Предстоящие ремонты»

  • Документ – дата, время создания и название документа, в котором запланировано выполнение ремонтных работ. 
  • Вид ремонтных работ — наименование вида обслуживания оборудования, по которому предстоит выполнить ремонтные работы.
  • Дата начала – плановая дата начала выполнения ремонтных работ.
  • Дата окончания – плановая дата окончания выполнения ремонтных работ.

 

Закладка «Журналы/ Сметы ремонта (Заявки на ремонт)»

  • Документ – дата, время создания и название заявки на выполнение ремонтных работ.
  • Дата начала ремонта – планируемая дата и время начала проведения заявленных работ.
  • Дата окончания ремонта — планируемая даты и время окончания проведения заявленных работ.
  • Гарантийный ремонт - признак ознает, что ремонт планируется выполнить в гарантийный период для объекта ремонта.

 

Закладка «Журналы/ Выполненные ремонты»

  • Ремонт завершен — признак завершения ремонта.
  • Документ — дата, время создания и название документа выполненного ремонта.
  • Вид ремонтных работ — наименование вида обслуживания оборудования, по которому выполнены ремонтные работы.
  • Гарантийный ремонт - признак ознает, что ремонт выполнен в гарантийный период для объекта ремонта.
  • Дата начала план – плановая дата начала выполнения ремонтных работ.
  • Дата окончания план – плановая дата окончания выполнения ремонтных работ.
  • Дата начала факт – дата начала выполнения ремонтных работ.
  • Дата окончания факт — дата окончания выполнения ремонтных работ.

 

Закладка «История перемещения оборудования»

  • Выбор вида иерархии — вид иерархии, по которой отображается история перемещения объекта ремонта.
  • № — номер документа в списке.
  • Дата — дата формирования документа.
  • Документ изменения — документ, которым оформлялось перемещение объекта ремонта.
  • Позиция — позиция объекта ремонта в иерархии.
